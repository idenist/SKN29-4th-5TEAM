## 1. 배포 아키텍처

```
React Frontend
      ↓
    Nginx (Reverse Proxy, Port 80)
      ↓
   Gunicorn (WSGI Server)
      ↓
   Django (MVT / DRF)
      ↓
  PostgreSQL Database
```

Nginx는 정적 파일 서빙 및 리버스 프록시 역할을 수행하며, Gunicorn이 Django 애플리케이션을 WSGI 서버로 구동한다.

---

## 2. 컨테이너 구성

`docker-compose.yml` 기준으로 아래 3개 서비스로 구성된다.

| 서비스 | 이미지/빌드 | 역할 |
|---|---|---|
| `db` | postgres:16-alpine | 데이터베이스 |
| `backend` | 커스텀 빌드 (Django + Gunicorn) | 애플리케이션 서버 |
| `nginx` | 커스텀 빌드 | 리버스 프록시, 정적 파일 서빙 |

각 서비스는 `restart: unless-stopped` 정책이 적용되어 컨테이너 비정상 종료 또는 호스트 재부팅 시 자동으로 재시작된다.

### 2.1 서비스 간 의존성

`backend` 서비스는 `db` 서비스의 헬스체크(`pg_isready`)가 정상 통과된 이후에 기동되도록 구성되어, 데이터베이스 준비 전 애플리케이션이 기동되는 문제를 방지한다.

### 2.2 데이터 영속성

| 볼륨 | 용도 |
|---|---|
| `postgres_data` | PostgreSQL 데이터 영속화 |
| `static_volume` | Django `collectstatic` 결과물을 Nginx와 공유 |

---

## 3. Django 컨테이너 (backend)

### 3.1 Dockerfile 구성

- Base 이미지: `python:3.11-slim`
- 빌드 시 PostgreSQL 클라이언트 라이브러리(`libpq-dev`), 빌드 도구(`build-essential`) 설치
- Python 의존성은 `requirements.txt` 기준으로 설치
- Gunicorn을 WSGI 서버로 사용
- 빌드 시 `collectstatic` 실행하여 정적 파일 수집

### 3.2 Gunicorn 실행

`gunicorn_config.py`를 통해 Gunicorn 설정을 관리하며, `config.wsgi:application`을 기동 대상으로 지정한다.

### 3.3 환경변수 관리

Django 설정은 `.env` 파일을 통해 관리되며, 데이터베이스 접속 정보, JWT 설정, CORS 허용 Origin, AWS 자격 증명 등이 환경변수로 분리되어 있다. `.env.example`을 통해 필요한 환경변수 항목을 문서화하였다.

---

## 4. Nginx 구성

Nginx는 80번 포트를 통해 외부 요청을 수신하며, 정적 파일 요청은 직접 서빙하고 애플리케이션 요청은 backend 컨테이너로 프록시한다.

---

## 5. CORS 및 프론트엔드 연동

React 프론트엔드와의 통신을 위해 Django 측에 `django-cors-headers`를 적용하였으며, 허용 Origin은 배포 환경에 따라 환경변수로 관리한다.

---

## 6. 데이터베이스 연결

로컬 개발 및 통합 테스트 환경에서는 컨테이너화된 PostgreSQL(`postgres:16-alpine`)을 사용하며, 운영 배포 환경에서는 AWS RDS 연결을 기준으로 하도록 설계되어 있다. 데이터베이스 접속 정보는 `.env`의 `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD` 환경변수를 통해 교체 가능한 구조이다.

---

## 7. AI/RAG 데이터 연동을 위한 볼륨 구성

AI 챗봇 기능에서 사용하는 벡터 데이터베이스(ChromaDB)는 별도 디렉토리(`data/vector_db`)에 파일 기반으로 저장되며, backend 컨테이너에 볼륨으로 마운트하여 컨테이너 환경에서도 동일하게 참조할 수 있도록 구성하였다.

| 설정 | 값 |
|---|---|
| 볼륨 마운트 | `./data/vector_db:/app/data/vector_db` |
| 환경변수 | `CHROMA_PERSIST_DIR=/app/data/vector_db` |
