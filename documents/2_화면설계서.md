## 4. 컴포넌트 레이아웃 아키텍처 (Layout Architecture)

### 4.1 기본 루트 레이아웃 (`RootLayout`)
- **역할:** 서비스 전체 페이지 패키징 및 글로벌 공통 컴포넌트 제어.
- **구조:** `Header` + `layout-main` (`PageContainer` > `Outlet`) + `Footer`
- **인터랙션 피드백 배너 기능:**
  - `useAuth` 훅 내 `sessionMessage` 상태 값 실시간 모니터링 수행.
  - 토큰 유효기간 만료 또는 강제 로그아웃 이벤트 수신 시 `layout-session-banner` 컴포넌트가 최상단에 슬라이드 렌더링 처리됨. "로그인 세션이 만료되었습니다. 다시 로그인해 주세요." 스크립트 출력 및 즉각적인 `/login` 라우트 인텐트 복귀 앵커 연동.

### 4.2 계정 인증 레이아웃 (`AuthLayout`)
- **역할:** 로그인, 회원가입, 패스워드 리셋 플로우 화면의 시각적 레이아웃 통일.
- **구조:** 독립 컴포넌트 스펙으로 구성, 상단 중앙 브랜드 로고 이미지(`home_logo.png`) 및 텍스트 앵커(`"이젠, 안쉼"`) 노출 후 `Outlet` 결합.

## 5. 화면별 상세 기능 명세 (Screen Specifications)

### 5.1 SCR-AUTH-001: 로그인 화면 (`LoginPage`)

#### [1] 화면 UI Mockup 및 레이아웃 구조
- `AuthCard` 컴포넌트를 코어로 채택하여 상단 키커 메시지(`"Welcome back"`), 타이틀 및 이메일/패스워드 입력 폼 수직 스택 배열.

#### [2] 컴포넌트 동작 및 이벤트 명세
- **Form 데이터 검증 규칙 (Validation Logic):**
  - 이메일 필드 트림(`trim()`) 처리 후 공백 검출 시 `errors.email = '이메일을 입력해 주세요.'` 메시지 표시.
  - 패스워드 필드 미입력 상태 검출 시 `errors.password = '비밀번호를 입력해 주세요.'` 메시지 표시.
- **비동기 API 트랜잭션 핸들링 및 예외 처리:**
  - `handleSubmit` 함수 트리거 시 `isSubmitting` 로컬 뷰 락을 `true`로 갱신하여 인풋 컴포넌트 및 서브밋 버튼을 즉시 비활성화(`disabled`) 처리하여 다중 서브밋(Double Submit) 이슈 차단.
  - 계정 데이터 불일치 에러 트래핑 시 Form 상단에 에러 문구 출력 가이드 처리.
  - 만일 백엔드 예외 코드가 `PASSWORD_RESET_REQUIRED`인 경우, 유저 조치 플로우 지원을 위해 `/forgot-password` 링크 버튼을 동적으로 생성 및 노출 처리.
- **인증 세션 연동:**
  - 로그인 통신 성공 시 `useAuth` 컨텍스트의 `login(tokens)` 함수를 호출하여 로컬 스토리지 내 `accessToken` 및 `refreshToken`을 실시간 동기화 바인딩 후 마이페이지 대시보드로 이동시킴.

### 5.2 SCR-POL-001: 청년 정책 검색 화면 (`PolicySearchPage`)

#### [1] 화면 UI 레이아웃 구성
- 좌측: 아사이드 사이드바 영역(`PolicyFilterPanel`) - 검색 조건 스크리닝 필터 타겟팅.
- 우측: 메인 검색 바(`SearchBar`), 검색 요약 정보 배너(`Toolbar`), 정책 카드 데이터 스트림 리스트(`PolicyList`), 하단 페이징 인디케이터(`Pagination`).

#### [2] 인터랙션 및 상태 제어 핵심 스펙
- **버퍼링 기반 조건 필터 동작 시스템 (`PolicyFilterPanel`):**
  - 분야(`['전체', '주거', '금융', '취업', '교육', '창업']`), 지역(`['전체', '서울', '경기', ...]`), 소득조건(`['전체', '제한없음', '중위소득', '저소득']`) 컬렉션 지원.
  - 사용자의 필터 조작 시 검색 쿼리의 무분별한 리프레시 스트레스를 제어하기 위해 로컬 `draftFilters` 객체 상태에 데이터를 버퍼링 기록함. 최종 '조건 적용' 버튼 터치 순간에 한하여 상위 상태 핸들러(`onChange`)로 서브밋함.
  - '초기화' 핸들러 동작 시 나이 입력을 빈 값으로 소거하고 분류 항목들은 `'전체'` 스트링으로 일괄 리셋 싱크 적용.
- **실시간 데이터 제어 배너 (`Toolbar`):**
  - 마감 공고 제외 체크박스 토글 감지 시 URL Query Parameter 스페이스에 `includeClosed=1` 인자를 유기적으로 밀어 넣거나 탈거 처리하여 서버사이드 필터링 동작을 직접 트리거함.
- **데이터 로딩 예외 시나리오 설계:**
  - 검색 조건 미충족 진입 시 기본 가이드 `EmptyState` 주입.
  - API 네트워크 지연 및 통신 장애 상황 발생 시 에러 메커니즘을 트래핑하여 컴포넌트 단에 `ErrorState` 노출 및 `refetch` 콜백 리트라이 버튼 제공.

### 5.3 SCR-POL-002: 정책 상세 정보 화면 (`PolicyDetailPage`)

#### [1] 화면 UI 레이아웃 구성
- 상단 네비게이션 복귀 링크 레이어(`"정책 목록으로"`) 배치.
- 헤더 영역 (`PolicyDetailHeader`): 모집 상태, 분야, 타겟 지자체 정보 뱃지 및 마감 타이머 스케줄 출력.
- 메인 바디 영역: 기본 정보 그리드(`PolicyInfoSection`), 신청 대상 요건 리스트(`PolicyRequirementSection`), 가이드 시퀀스(`PolicyApplySection`), 하단 연관 컴포넌트 목록(`RelatedPolicyList`).

#### [2] 컴포넌트 동작 및 비즈니스 로직
- **인터랙티브 스크랩 토글 및 예외 경로 제어 (`ScrapButton`):**
  - 미인증 유저가 스크랩 액션 발생 시 통신을 차단하고 `scrapMessage` 상태에 `"로그인이 필요한 기능입니다."`를 주입하여 동적 인라인 알림 처리.
  - 인증 유저 액션 시 낙관적 업데이트(Optimistic Update) 설계 기법을 반영하여 UI 인디케이터 스크랩 활성화 상태(`isScrapped`)를 우선적으로 반전 변환하고, 백엔드 서버 엔드포인트 비동기 API(`createScrap` / `deleteScrap`) 실행. 실패 케이스 발생 시 캐치 블록 내에서 상태 스냅샷을 이전 값으로 정상 롤백 처리 후 시스템 예외 텍스트 출력.
- **신청 가이드 외부 채널 연동 아키텍처 (`PolicyInfoSection`):**
  - 지원 내용, 연령 조건, 소득 조건, 문의처 등 비즈니스 코어 데이터를 가로 2열 테이블 형태로 맵핑 구조화함.
  - 백엔드 스키마 데이터 파싱 결과를 상시 검증하여, 공식 접수처 URL(`applyUrl`) 데이터가 공백이거나 유효하지 않은 포맷인 경우 "신청하러 가기" 버튼을 강제로 `disabled` 마킹 처리하고 레이블 텍스트를 "신청 링크 준비 중"으로 가변 대체 적용함.

### 5.4 SCR-CHAT-001: AI 안쉼 챗봇 화면 (`ChatPage`)

#### [1] 화면 UI 레이아웃 구성
- 탑 라인: AI 챗봇 안내 세션 헤더 노출.
- 센터 스플릿 좌측 영역: AI 대화 메인 스레드 윈도우(`ChatWindow`), 하단 대화 입력 폼 컴포넌트(`ChatInput`).
- 센터 스플릿 우측 패널 영역: AI 컨텍스트가 추천하는 실시간 청년 정책 리스트 가이드 보드(`RecommendedPolicyList`).

#### [2] 데이터 통신 연동 및 인텔리전스 예외 처리 스펙
- **유저 프로필 컨텍스트 지능형 주입 처리 (`buildUserProfile`):**
  - 무맥락 대화로 인한 응답 품질 저하 및 인프라 비용 누수를 방지하기 위해 유저 인증 세션 상태(`user.profile`) 정보를 가로채어 `age`, `region`, `interests` 파라미터가 포함된 사용자 프로필 객체를 자동 가공함. AI RAG 엔진 API 파이프라인 송신 시 페이로드 구조 내에 캡슐화 처리하여 동시 전송 처리함.
- **출처 및 경고 컴포넌트 지원 영역 명세 (발표 피드백 반영):**
  - 생성형 AI 모델 특유의 할루시네이션(Hallucination) 현상을 방지하고 데이터 투명성을 확보하기 위해, 응답 데이터 인터페이스 사양에 맞춰 수신된 출처 데이터(`response.sources`) 및 안내 경고 주의문(`response.warnings`)을 메시지 버블 하단에 컴포넌트 서포팅 레이어 형식으로 완벽하게 파싱 렌더링 처리함.
- **채팅 입력 인터랙션 처리 명세 (`ChatInput`):**
  - 엔터키(`Enter`) 트리거 시 폼 서브밋 함수(`submit()`)를 호출하되, `Shift + Enter` 조합 키 입력 발생 시에는 멀티라인 개행 처리가 이루어지도록 이벤트를 분기 바인딩 제어함.
  - API 비동기 처리 트래픽이 유지되는 동안에는 타이핑 인디케이터 애니메이션 컴포넌트(`TypingIndicator`)를 활성화 출력하여 유저 이탈을 예방함.

### 5.5 SCR-MY-001: 마이페이지 메인 화면 (`MyPage`)

#### [1] 화면 UI 레이아웃 구성
- 상단 영역: 프로필 요약 카드(`ProfileCard`)와 사용자 활동 계량 메트릭스 대시보드(`MyPageSummary`)를 1:1 그리드 스페이스로 평행 구조 정렬.
- 하단 영역: 활동 내역 멀티 탭 보드 패널(`MyPolicyTabs`) 및 미확인 실시간 알림 피드(`NotificationPreviewList`) 배치.

#### [2] 컴포넌트 기능 구현 세부 사항
- **통합 요약 대시보드 데이터 바인딩:**
  - `MyPageSummary` 컴포넌트 진입 시 백엔드 스키마 응답 데이터 구조인 `summary` 스냅샷을 기반으로 스크랩 총 개수(`scrapsCount`), 최근 본 정책(`viewedCount`), 누적 검색어 수(`searchCount`), 미독 알림 개수(`unreadNotifications`) 데이터를 개별 카운트 모듈에 정밀 맵핑함.
- **로컬 탭 뷰 스위칭 아키텍처 (`MyPolicyTabs`):**
  - `activeTab` 전용 인메모리 상태 인덱스를 운용하여 사용자가 선택한 카테고리(`scraps` / `viewed` / `history`)에 따라 하부 독립 컴포넌트 데이터 스트림 레이어를 동적으로 스왑 렌더링하도록 흐름을 제어함.
- **알림 미리보기 뷰 필터링 제어 (`NotificationPreviewList`):**
