# [이젠, 안쉼] 4차 프로젝트 화면설계서

## 💡 주요 평가 항목 반영 요약 (Executive Summary)
본 화면설계서는 프로젝트의 핵심 평가 지표를 완벽히 충족하도록 설계되었습니다.
1. **반응형 UI 설계:** 전 화면에 걸쳐 CSS Flex/Grid 레이아웃을 채택하여 PC, 태블릿, 모바일에 최적화된 유연한 화면을 제공합니다.
2. **화면-구조 매핑 정합성:** React 프론트엔드의 각 화면은 Django 백엔드의 MVT(Model-View-Template/API) 아키텍처 및 요구사항 ID와 누락 없이 1:1로 매핑됩니다.
3. **인증 흐름 및 권한 설계:** JWT 토큰 기반으로 화면 단위 및 컴포넌트 단위(버튼 노출 여부 등)의 정교한 접근 제어(RBAC)가 적용되었습니다.
4. **LLM 상호작용 UX 설계:** 비동기 API 호출 시 발생할 수 있는 모든 상태(입력 대기 ➔ 로딩/스트리밍 ➔ 응답 성공 ➔ 예외/오류)에 대한 UX 처리를 명시했습니다.

---

## 1. 메인 화면
* **화면 목적:** 사용자가 사이트 접속 시 맞춤형 정책을 확인하고 주요 메뉴로 진입할 수 있도록 안내
* **요구사항 & MVT 매핑:** [REQ-F-03, REQ-F-04] / Django `PolicyListView` API 연동하여 추천 데이터 서빙
* **인증 및 접근 권한:** [Public] 누구나 접근 가능. 단, 로그인 유저에게는 DB(UserProfile) 기반 맞춤형 정책을 렌더링하고, 비로그인 유저에게는 조회수 기반 인기 정책 렌더링
* **비동기 UX 및 예외 상태:** 맞춤 정책 데이터를 불러오는 동안 Card Skeleton UI(로딩 상태) 노출 / 데이터가 없을 경우 "프로필을 완성하고 맞춤 정책을 받아보세요" 빈 상태(Empty State) 표시
* **반응형 설계 (Flex/Grid):** 정책 카드 리스트를 `CSS Grid`로 구성. PC(3열, `grid-template-columns: repeat(3, 1fr)`) ➔ 태블릿(2열) ➔ 모바일(1열 가로 스와이프)로 가변 반응형 적용

## 2. 회원가입/로그인 화면
* **화면 목적:** 사용자 인증(JWT) 및 개인화 추천을 위한 필수 프로필 정보 수집
* **요구사항 & MVT 매핑:** [REQ-F-01] / Django Auth, `User` 및 `UserProfile` 모델 MVT 완벽 매핑
* **인증 및 접근 권한:** [Anonymous Only] 이미 로그인된 유저가 접근 시 메인 화면으로 강제 리다이렉트(Redirect) 처리
* **비동기 UX 및 예외 상태:** 로그인/가입 API 비동기 요청 중 제출 버튼 비활성화(Disabled) 및 스피너 표시 / 아이디 중복, 비밀번호 양식 오류 시 폼 하단에 즉각적인 Red 텍스트 경고 노출
* **반응형 설계 (Flex/Grid):** 중앙 정렬 `Flexbox` 사용. 모바일 기기 키보드 활성화 시 입력창이 가려지지 않도록 컨테이너 스크롤 및 여백(Padding-bottom) 동적 조절

## 3. 정책 검색 화면
* **화면 목적:** 키워드와 다중 조건 필터를 활용하여 전체 청년 정책을 탐색
* **요구사항 & MVT 매핑:** [REQ-F-04] / Django `SearchHistory` 모델 기반 통계 처리 및 `FilterView` API 연동
* **인증 및 접근 권한:** [Public] 누구나 접근 가능
* **비동기 UX 및 예외 상태:** 검색 API 호출 시 결과 영역에 로딩 스피너 표시 / 조건에 맞는 결과가 없을 시 "조건을 변경하여 다시 시도해 주세요" 일러스트 표출
* **반응형 설계 (Flex/Grid):** PC에서는 좌측 필터 바(Aside)와 우측 검색 결과(Grid)의 2-Column 레이아웃 구성. 모바일에서는 필터 바를 숨기고 '필터' 버튼 클릭 시 하단 모달(BottomSheet)로 띄우도록 구조 변경

## 4. 정책 상세 화면
* **화면 목적:** 선택한 정책의 상세 정보 및 신청 링크, 최대 예상 지원금을 시각적으로 제공
* **요구사항 & MVT 매핑:** [REQ-F-06, REQ-F-07] / Django `Policy`, `Scrap` 모델 매핑
* **인증 및 접근 권한:** [Mixed] 상세 조회는 누구나 가능하나, **'스크랩(찜)' 버튼과 '예상 지원금 계산기'는 로그인 유저(Authenticated)만 사용 가능** (비로그인 유저 클릭 시 로그인 페이지 모달 호출)
* **비동기 UX 및 예외 상태:** 스크랩 버튼 클릭 시 비동기 처리로 화면 새로고침 없이 즉각적인 토글(★) 애니메이션 피드백 제공
* **반응형 설계 (Flex/Grid):** PC에서는 우측에 플로팅 박스(`position: sticky`)로 신청/스크랩 버튼 배치. 모바일 환경에서는 사용성을 위해 화면 하단 고정 바(Fixed Bottom Bar)로 전환

## 5. AI 챗봇 화면 (LLM UX 핵심)
* **화면 목적:** 자연어 질의를 통해 LLM 기반 맞춤형 정책을 대화형으로 탐색
* **요구사항 & MVT 매핑:** [REQ-F-05, REQ-N-02] / RAG 파이프라인 연동, Django `ChatAPIView` 비동기 매핑
* **인증 및 접근 권한:** [Public] 허용하되, 로그인 유저의 경우 `UserProfile` 데이터를 프롬프트 컨텍스트에 자동 포함하여 개인화 답변 제공
* **비동기 UX 및 예외 상태 (UX 집중 설계):** 
  - **입력 상태:** 질문 입력 시 전송 버튼 활성화.
  - **대기/로딩 상태:** API 호출 즉시 입력창 비활성화, 채팅창에 "에이전트가 정책을 검색 중입니다..." 타이핑 효과 및 로딩 스피너 노출.
  - **예외 상태 (타임아웃/환각 방지):** 15초 이상 지연 시 HTTP 408 처리 및 "잠시 후 다시 시도해주세요" 버튼 렌더링. 출처(URL)가 없으면 "원문 확인이 필요합니다" 경고 라벨 부착.
* **반응형 설계 (Flex/Grid):** `Flex-direction: column`으로 채팅 내역(가변 영역)과 하단 입력창(고정 영역) 분리. 모바일 가상 키보드 호출 시 화면이 밀려 올라가도록 뷰포트(`100dvh`) 최적화.

## 6. 마이페이지 (정보 대시보드)
* **화면 목적:** 사용자의 프로필, 스크랩, 최근 본 공고, 예상 혜택 요약을 통합 관리
* **요구사항 & MVT 매핑:** [REQ-F-02, REQ-F-07] / Django `UserProfile`, `ViewedPolicy` 등 유저 관련 모델 총합 API 매핑
* **인증 및 접근 권한:** [Private - Authenticated Only] 미인증 유저 접근 시 401 권한 에러 처리 및 로그인 화면으로 즉시 리다이렉트
* **비동기 UX 및 예외 상태:** 각 활동 이력(위젯)별로 비동기 호출을 분리하여, 하나의 API가 늦어도 다른 위젯은 먼저 화면에 렌더링되도록 병렬 처리(Promise.all) 및 에러 바운더리 적용
* **반응형 설계 (Flex/Grid):** 4개의 주요 위젯을 `CSS Grid`를 활용해 PC(2x2 배열), 태블릿/모바일(1x4 세로 배열)로 유동적 재배치

## 7. 알림 화면
* **화면 목적:** 스크랩한 공고의 마감 임박 및 내용 변경 사항을 사용자에게 알림
* **요구사항 & MVT 매핑:** [REQ-F-06] / Django `Notification` 모델 매핑 (Celery 스케줄러 기반)
* **인증 및 접근 권한:** [Private - Authenticated Only] 본인(`request.user`) 소유의 알림 데이터만 조회 및 삭제 가능하도록 인가(Authorization) 로직 적용
* **비동기 UX 및 예외 상태:** 개별 알림 '읽음' 처리 및 '삭제' 시 비동기로 리스트에서 즉각적으로 사라지는(Fade-out) 모션 적용. 알림이 없을 경우 빈 상태 UI 표시.
* **반응형 설계 (Flex/Grid):** `Flex` 기반의 리스트 아이템 디자인. 텍스트 오버플로우 발생 시 `text-overflow: ellipsis`를 적용하여 레이아웃 깨짐 방지.

## 8. 뉴스 화면 및 9. 영상 화면
* **화면 목적:** 청년 정책과 관련된 외부 미디어 정보(뉴스/유튜브) 제공
* **요구사항 & MVT 매핑:** [REQ-F-08] / 외부 네이버 News API 및 YouTube iframe Embed 컴포넌트 매핑
* **인증 및 접근 권한:** [Public] 누구나 접근 가능
* **비동기 UX 및 예외 상태:** 외부 API 통신 장애 발생 시 전체 화면이 다운되지 않도록 예외 처리(Fallback UI)를 적용하여 "현재 미디어 정보를 불러올 수 없습니다"라는 대체 컴포넌트 노출
* **반응형 설계 (Flex/Grid):** 뉴스는 `CSS Grid`로 다단 카드 레이아웃 구성. 유튜브 영상은 `aspect-ratio: 16/9` 속성을 부여하여 디바이스 가로폭에 맞춰 세로 비율이 자동 조정되는 반응형 적용.

## 10. 커뮤니티 화면
* **화면 목적:** 청년들 간의 정책 후기 및 정보 공유 게시판 제공
* **요구사항 & MVT 매핑:** [REQ-F-09] / Django `CommunityPost`, `Comment` 모델 기반 CRUD API 매핑
* **인증 및 접근 권한:** [Mixed] 목록 및 상세글 조회는 [Public]. **글쓰기, 수정, 삭제는 [Authenticated] 로그인 필수**이며, 수정/삭제는 본인이 작성한 글에만 권한(IsOwner) 부여.
* **비동기 UX 및 예외 상태:** 글 작성/수정 완료 시 로딩 인디케이터 표시 후 목록으로 부드럽게 화면 전환(Client-side Routing).
* **반응형 설계 (Flex/Grid):** 화면 크기에 따라 PC에서는 번호, 제목, 작성일이 분리된 테이블(`Table`/`Grid`) 형태 제공, 모바일에서는 텍스트 중심의 세로형 카드 리스트(`Flex`) 형태로 UI 180도 전환.

## 11. 개인정보처리방침 (Footer)
* **화면 목적:** 사용자 정보 수집 및 이용 목적을 고지하여 규정 준수
* **요구사항 & MVT 매핑:** [REQ-N-05] / React 정적 라우팅(`PrivacyPolicyPage`)
* **인증 및 접근 권한:** [Public] 누구나 열람 가능
* **반응형 설계 (Flex/Grid):** 긴 텍스트 문서이므로 모바일 환경에서 가독성을 해치지 않도록 컨테이너 좌우 여백(Padding)을 디바이스에 맞게 축소하고, 줄간격(Line-height)을 1.6 이상으로 최적화.
