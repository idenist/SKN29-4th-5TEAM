# [이젠, 안쉼] 4차 프로젝트 화면설계서

## 문서 개요
본 문서는 React 프론트엔드 기반의 주요 화면 흐름과 UI 요소, 반응형 고려사항을 정리한 화면설계서입니다. 화면별 예외 상태(로딩/오류) 처리와 요구사항 연결성을 명시합니다.

---

## 1. 메인 화면[cite: 3]
* **화면 목적:** 사용자가 사이트 접속 시 맞춤형 정책을 확인하고 주요 메뉴로 진입할 수 있도록 안내[cite: 3]
* **주요 컴포넌트:** 헤더(GNB), 메인 배너, 개인화 추천 정책 카드 리스트, 실시간 검색어 순위표[cite: 3]
* **연결 요구사항 ID:** REQ-F-03, REQ-F-04, REQ-N-01[cite: 3]
* **API 연동 여부:** 맞춤형 추천 API, 실시간 검색어 집계 API 연동[cite: 3]
* **로딩/오류/빈 상태:** 추천 정책 로딩 중 스피너 표시 / 맞춤 데이터가 없을 경우 "프로필을 완성하고 맞춤 정책을 받아보세요" 안내 표시[cite: 3]
* **반응형 고려:** 정책 카드 배치는 PC 3열, 태블릿 2열, 모바일 1열(가로 스와이프)로 구성[cite: 3]

## 2. 회원가입/로그인 화면[cite: 3]
* **화면 목적:** 사용자 인증(JWT) 및 맞춤형 추천을 위한 프로필(연령, 거주지, 관심분야) 수집[cite: 3]
* **주요 컴포넌트:** ID/PW 입력폼, 연령/거주지/관심분야 선택 드롭다운, 개인정보처리방침 동의 체크박스, 제출 버튼[cite: 3]
* **연결 요구사항 ID:** REQ-F-01, REQ-N-03[cite: 3]
* **API 연동 여부:** 로그인 API(토큰 발급), 회원가입 API (Form Validation 수행)[cite: 3]
* **로딩/오류/빈 상태:** 아이디 중복, 비밀번호 양식 오류 시 입력창 하단에 빨간색 오류 텍스트 표시[cite: 3]
* **반응형 고려:** 모바일 기기 키보드 활성화 시 입력창이 가려지지 않도록 화면 스크롤 처리[cite: 3]

## 3. 정책 검색 화면[cite: 3]
* **화면 목적:** 키워드와 조건 필터를 활용하여 전체 청년 정책을 탐색[cite: 3]
* **주요 컴포넌트:** 검색바, 카테고리 필터(지역, 연령, 분야, 마감 상태), 검색 결과 목록(카드형), 페이지네이션[cite: 3]
* **연결 요구사항 ID:** REQ-F-04[cite: 3]
* **API 연동 여부:** 정책 목록/검색 API 연동[cite: 3]
* **로딩/오류/빈 상태:** 검색 결과 로딩 스피너 표시 / 조건에 맞는 정책이 없을 시 "조건을 변경하여 다시 시도해 주세요" 빈 상태 렌더링[cite: 3]
* **반응형 고려:** 모바일 화면에서는 필터 영역을 상단 버튼으로 숨기고 클릭 시 모달(BottomSheet) 형태로 제공[cite: 3]

## 4. 정책 상세 화면[cite: 3]
* **화면 목적:** 선택한 정책의 상세 정보 및 신청 링크, 출처를 제공하고 스크랩 기능 지원[cite: 3]
* **주요 컴포넌트:** 정책 타이틀, 지원 내용 및 대상 요약, 예상 지원금 카드, 스크랩(찜) 버튼, 원문/신청 URL 이동 버튼[cite: 3]
* **연결 요구사항 ID:** REQ-F-06, REQ-F-07[cite: 3]
* **API 연동 여부:** 정책 상세 API, 스크랩 추가/취소 API 연동[cite: 3]
* **로딩/오류/빈 상태:** 비로그인 유저가 스크랩 버튼 클릭 시 "로그인 후 이용 가능합니다" 알럿(Alert) 노출[cite: 3]
* **반응형 고려:** PC에서는 우측에 플로팅 상태로 스크랩 및 신청 버튼 배치, 모바일에서는 화면 하단 고정 바(Fixed Bottom Bar)로 배치[cite: 3]

## 5. AI 챗봇 화면[cite: 3]
* **화면 목적:** 자연어 질의를 통해 LLM 기반 맞춤형 정책을 추천받고 대화형으로 정보 탐색[cite: 3]
* **주요 컴포넌트:** 대화창(채팅 버블), 질문 입력 필드, 전송 버튼, 추천 정책 미니 카드, 출처 이동 버튼[cite: 3]
* **연결 요구사항 ID:** REQ-F-05, REQ-N-02[cite: 3]
* **API 연동 여부:** `/api/ai/chat/` 비동기(Async) Fetch 연동[cite: 3]
* **로딩/오류/빈 상태:** API 대기 시 "에이전트 검색 중" 로딩 스피너 표시 / 타임아웃 또는 실패 시 "AI 답변 생성에 실패했습니다. 잠시 후 시도해주세요" 오류 메시지와 재시도 버튼 표시[cite: 3]
* **반응형 고려:** 모바일 환경에서 챗봇 입력창을 하단 고정하고 화면 키보드 호출 시 자연스럽게 위로 올라오도록 설계[cite: 3]

## 6. 마이페이지[cite: 3]
* **화면 목적:** 사용자의 개인화된 정보(프로필, 활동 이력, 예상 혜택)를 통합 관리[cite: 3]
* **주요 컴포넌트:** 프로필 이미지 업로드 영역, 내 정보 요약, 최대 예상 지원금, 스크랩 공고 목록, 최근 본 공고, 검색 기록[cite: 3]
* **연결 요구사항 ID:** REQ-F-02, REQ-F-07[cite: 3]
* **API 연동 여부:** 마이페이지 통합 API, 프로필 사진 S3 업로드 API[cite: 3]
* **로딩/오류/빈 상태:** 비로그인 접근 시 로그인 화면으로 강제 리다이렉트 / 스크랩 내역이 없으면 빈 상태 아이콘 표출[cite: 3]
* **반응형 고려:** 각 활동 이력 위젯을 PC에서는 Grid 모듈로 분할 배치, 모바일에서는 세로 스크롤 리스트로 변환[cite: 3]

## 7. 알림 화면[cite: 3]
* **화면 목적:** 스크랩한 공고의 마감 임박 및 내용 변경 사항을 사용자에게 알림[cite: 3]
* **주요 컴포넌트:** 알림 리스트(마감 7일/3일 전, 변경 사항), 읽음/안읽음 표시, 개별 알림 삭제 버튼[cite: 3]
* **연결 요구사항 ID:** REQ-F-06[cite: 3]
* **API 연동 여부:** 알림 목록 조회 및 읽음 처리 API 연동[cite: 3]
* **로딩/오류/빈 상태:** 수신된 알림이 없으면 "새로운 알림이 없습니다" 표시[cite: 3]
* **반응형 고려:** 알림 제목이 길 경우 모바일에서는 말줄임표(...) 처리[cite: 3]

## 8. 뉴스 화면[cite: 3]
* **화면 목적:** 청년 정책과 관련된 외부 뉴스 정보를 카드 형태로 제공[cite: 3]
* **주요 컴포넌트:** 뉴스 목록 카드(기사 제목, 요약, 언론사, 발행일), 원문 보러가기 버튼[cite: 3]
* **연결 요구사항 ID:** REQ-F-08[cite: 3]
* **API 연동 여부:** 네이버 뉴스 API 데이터 기반 조회[cite: 3]
* **로딩/오류/빈 상태:** 외부 API 호출 실패 시 화면 깨짐 없이 "뉴스 정보를 불러오지 못했습니다. 잠시 후 다시 시도해주세요." 예외 처리 UI 표시[cite: 3]
* **반응형 고려:** PC 3열, 태블릿 2열, 모바일 1열 카드뷰[cite: 3]

## 9. 영상 화면[cite: 3]
* **화면 목적:** 청년 정책 관련 유튜브 안내 영상을 시각적으로 제공[cite: 3]
* **주요 컴포넌트:** 영상 카드(유튜브 iframe 임베딩, 제목, 설명, 출처)[cite: 3]
* **연결 요구사항 ID:** REQ-F-08[cite: 3]
* **API 연동 여부:** YouTube iframe Embed 적용[cite: 3]
* **로딩/오류/빈 상태:** 영상 로드 실패/삭제 시 "영상을 불러오지 못했습니다. 출처를 확인해주세요." 대체 문구 표시. 자동 재생 금지[cite: 3]
* **반응형 고려:** 고정 높이 픽셀 사용을 지양하고 iframe에 `aspect-video` (16:9) 반응형 비율 적용[cite: 3]

## 10. 커뮤니티 화면[cite: 3]
* **화면 목적:** 청년들 간의 정책 후기 및 정보 공유 게시판 제공 (MVP 수준)[cite: 3]
* **주요 컴포넌트:** 게시글 목록, 게시글 텍스트 카드, 글쓰기 플로팅 버튼(FAB)[cite: 3]
* **연결 요구사항 ID:** REQ-F-09[cite: 3]
* **API 연동 여부:** 커뮤니티 게시글 목록/작성 API[cite: 3]
* **로딩/오류/빈 상태:** 비로그인 사용자가 글쓰기 버튼 클릭 시 "해당 기능은 로그인 후 이용할 수 있습니다" 로그인 유도 알럿 표시[cite: 3]
* **반응형 고려:** PC에서는 테이블 형태, 모바일에서는 텍스트 중심의 카드 리스트 형태로 UI 변경[cite: 3]

## 11. 개인정보처리방침 화면 또는 Footer 영역[cite: 3]
* **화면 목적:** 사용자 정보 수집 및 이용 목적을 고지하여 규정 준수[cite: 3]
* **주요 컴포넌트:** 하단 Footer 내 링크, 클릭 시 열리는 개인정보처리방침 모달(또는 별도 페이지)[cite: 3]
* **연결 요구사항 ID:** REQ-N-05[cite: 3]
* **API 연동 여부:** 없음 (정적 텍스트 렌더링)[cite: 3]
* **로딩/오류/빈 상태:** 해당사항 없음 (항시 노출)[cite: 3]
* **반응형 고려:** 문구가 길어지므로 모바일 환경에서는 텍스트 컨테이너 내부에 독립적인 세로 스크롤 적용[cite: 3]
