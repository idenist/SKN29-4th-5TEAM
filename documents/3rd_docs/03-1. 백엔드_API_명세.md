> SK네트웍스 Family AI 캠프 29기 3rd Team5 | FastAPI 백엔드 | 2026.06.15  
> Swagger UI: `http://localhost:8000/docs`

## 0. 문서 목적

본 문서는 Streamlit 프론트엔드, FastAPI 백엔드, RAG 파이프라인이 동일한 응답 계약을 기준으로 연동될 수 있도록 API 입력/출력 형식을 정리한 문서이다.

이번 최종본에는 다음 수정 사항을 반영하였다.

| 구분 | 반영 여부 | 내용 |
|---|---:|---|
| 1순위 | 반영 | `/api/chat`의 `recommendations[]`에 `application_url`이 보존되도록 명세 |
| 2순위 | 반영 | `item_id/title`을 우선 필드로 사용하고 `policy_id/policy_name`을 alias로 함께 제공 |
| 3순위 | 코드 미수정, 문서 반영 | `/api/policies`는 현재 코드 기준인 `limit/offset` 방식으로 명세 |
| 4순위 | 반영 | `deadline_status` 기본값과 마감 필터링 기준을 `startup_notice` 중심으로 정리 |
| 5순위 | 미반영 | `UserConditions.region_code`, `UserConditions.keywords`는 선택 확장 항목으로 분리 |

기존 프로젝트가 청년정책 중심에서 `policy`, `startup_notice`, `training`을 함께 다루는 통합 지원 정보 탐색 구조로 확장되었기 때문에, 본 문서에서는 `/api/policies`라는 기존 엔드포인트명을 유지하되 실제 응답 대상은 다음 세 가지 지원 항목 전체로 정의한다.

| source_category | 화면 표시명 | 설명 |
|---|---|---|
| `policy` | 청년정책 | 온통청년 기반 정책 정보 |
| `startup_notice` | 창업지원 | K-Startup 등 창업지원 공고 |
| `training` | 교육·훈련 | HRD/고용24 기반 교육훈련 과정 |

### 1.1 공통 응답 규칙

| 규칙 | 내용 |
|---|---|
| 단일 식별자 | 시스템 전체 연결 key는 `item_id`이다. |
| 기존 호환 alias | 기존 정책 중심 코드 호환을 위해 `policy_id = item_id`, `policy_name = title`을 함께 내려준다. |
| 제목 필드 | 화면 표시명은 `title`을 우선 사용한다. 기존 호환 필드는 `policy_name`이다. |
| 신청 버튼 | `application_url`이 `null` 또는 빈 문자열이면 신청 버튼을 숨긴다. |
| 출처 버튼 | `source_url`이 있으면 출처 보기 링크로 표시한다. |
| 상세 확인 | `needs_detail_check=true`이면 원문 확인 필요 문구를 함께 표시한다. |
| 완성도 점수 | `info_score`는 데이터 완성도 점수이며, 사용자 적합도 또는 추천 점수가 아니다. |
| 임의 생성 금지 | 원본에 없는 신청방법, 제출서류, 자격조건, 금액, 기간은 생성하지 않는다. |
| 결측치 처리 | API에서는 `null` 또는 빈 문자열이 내려올 수 있다. 프론트에서는 둘 다 결측으로 보고 숨김 처리한다. |

### 1.2 HTTP 상태 코드

| 코드 | 의미 | 발생 조건 |
|---|---|---|
| 200 | 성공 | 정상 응답 |
| 400 | 잘못된 요청 | 메시지 2자 미만 등 명시적 요청 오류 |
| 404 | 찾을 수 없음 | 존재하지 않는 `item_id` 상세 조회 |
| 422 | 유효성 오류 | Pydantic 검증 실패, 필드 누락, 타입 오류 |
| 500 | 서버 내부 오류 | 예상치 못한 예외 |
| 503 | 서비스 불가 | 데이터 파일 없음, 서버 메모리 부족, Vector DB 준비 안 됨 |

### 1.3 오류 응답 포맷

일반 오류:

```json
{
  "detail": "오류 메시지"
}
```

Pydantic 검증 오류:

```json

  "detail": "요청 형식이 올바르지 않습니다.",
  "errors": []
}
```

## 5. RAG 질의응답

### `POST /api/chat`

자연어 질문을 입력하면 RAG workflow가 조건 추출, 라우팅, 검색, 자격 가능성 판단, 답변 생성을 수행하고 최종 답변과 추천 항목을 반환한다.

### 5.1 내부 처리 흐름

```text
사용자 메시지 입력
→ 입력 검증
→ Condition Extractor
   - 나이, 지역, 소득, 고용 상태, 관심 도메인 추출
→ Router
   - route: 주거/교육/창업/금융/일자리 등 질의 도메인 결정
   - source_category: policy/startup_notice/training 결정
→ Retriever
   - Chroma Vector DB에서 관련 chunk 검색
   - item_id 기준 중복 제거
   - source_category/domain/마감 상태 기반 재정렬
→ Eligibility Checker
   - 사용자 조건과 지원 항목 조건 비교
→ Answer Generator
   - 원문 근거 기반 답변 생성
→ ChatResponse 반환
```

### 5.2 요청 바디 `ChatRequest`

```json
{
  "message": "서울에 사는 27세 취업준비생인데 받을 수 있는 청년 지원 정책 알려줘",
  "user_profile": {
    "age": 27,
    "region": "서울",
    "income": null,
    "employment_status": "취업준비생",
    "interest_domain": "일자리"
  },
  "top_k": 5
}
```

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `message` | `string` | 필수 | 사용자 자연어 질문. 2자 미만이면 400 반환 |
| `user_profile` | `object | null` | 선택 | Streamlit 사이드바 등에서 입력한 사전 조건 |
| `user_profile.age` | `int | null` | 선택 | 사용자 나이 |
| `user_profile.region` | `string | null` | 선택 | 거주 지역 |
| `user_profile.income` | `string | null` | 선택 | 소득 조건 |
| `user_profile.employment_status` | `string | null` | 선택 | 고용 상태 |
| `user_profile.interest_domain` | `string | null` | 선택 | 관심 도메인 |
| `top_k` | `int` | 선택 | 최종 추천 항목 수. 기본 5, 허용 범위 1~20 |

> 현재 코드에서 `user_profile` 병합 로직이 제한적일 수 있으므로, 실제 조건 판단은 `message`에서 추출된 값이 우선 사용될 수 있다.

### 5.4 최상위 응답 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `answer` | `string` | LLM 또는 fallback generator가 생성한 최종 답변 |
| `user_conditions` | `object` | 조건 추출 결과 |
| `route` | `string` | Router가 판단한 질의 도메인. 예: `주거`, `교육`, `창업`, `금융`, `일자리`, `전체` |
| `route_reason` | `string | null` | 라우팅 판단 이유 |
| `recommendations` | `array` | 추천 지원 항목 목록 |
| `warnings` | `array[string]` | fallback, 원문 확인 필요, 검색 결과 부족 등 경고 메시지 |
| `tool_trace` | `array[object]` | ReAct/도구 라우팅 디버그 이력 |
| `internal_search_sufficient` | `boolean` | 내부 Vector DB 검색 결과 충분성 판단 |
| `sufficiency_reasons` | `array[string]` | 검색 결과 충분성 판단 근거 |
| `next_action` | `string` | 충분성 판단 이후 선택된 다음 행동 |
| `external_used` | `boolean` | 외부 공식 출처 검색 도구 사용 여부 |
| `external_search_status` | `string` | 외부 공식 출처 검색 상태 |
| `external_search_targets` | `array[string]` | 외부 검색 대상으로 선택된 공식 출처 목록 |
| `external_search_queries` | `array[string]` | 외부 공식 출처 검색어 후보 |

> `source_category`, `errors`, `session_id`, `region_code`, `keywords`는 현재 필수 응답 계약에 포함하지 않는다. 추후 5순위 확장을 진행하면 별도 보강한다.

### 5.5 `user_conditions` 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `age` | `int | null` | 추출된 나이 |
| `region` | `string | null` | 추출된 지역명 |
| `income` | `string | null` | 소득 조건 |
| `employment_status` | `string | null` | 고용 상태 |
| `interest_domain` | `string | null` | 사용자 관심 분야 |

### 5.6 `recommendations[]` 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `item_id` | `string` | 통합 지원 항목 고유 ID. 프론트/백엔드/RAG 연결 기준 key |
| `title` | `string` | 화면 표시용 항목명 |
| `policy_id` | `string` | 기존 코드 호환용 ID. `item_id`와 동일하게 유지 |
| `policy_name` | `string` | 기존 코드 호환용 항목명. `title`과 동일하게 유지 |
| `source_category` | `string` | `policy`, `startup_notice`, `training` |
| `domain` | `string | null` | 분야 또는 라우팅 도메인 |
| `summary` | `string | null` | 요약 설명 |
| `eligibility` | `string` | `가능성 높음`, `추가 확인 필요`, `가능성 낮음` |
| `score` | `float` | 검색 유사도 점수. 사용자 적합도 점수로 표시하지 않는 것을 권장 |
| `reason` | `string` | 추천 사유 |
| `support_content` | `string | null` | 지원 내용 |
| `application_period` | `string | null` | 신청/접수/훈련 기간 원문 |
| `required_documents` | `string | null` | 제출 서류 |
| `source_url` | `string | null` | 원문 출처 URL |
| `application_url` | `string | null` | 신청 URL. 없으면 신청 버튼 숨김 |
| `needs_detail_check` | `boolean` | 상세 원문 확인 필요 여부 |
| `cautions` | `array[string]` | 주의사항 및 불충족 가능 조건 |
| `deadline_status` | `string` | `open`, `expired`, `unknown` |
| `application_end_date` | `string | null` | 파싱된 신청/접수 종료일. `YYYY-MM-DD` |
| `is_expired` | `boolean` | 마감 여부 |

### 5.7 마감 상태 처리 기준

| source_category | deadline_status 기준 |
|---|---|
| `startup_notice` | 신청/접수 종료일을 파싱하여 `open`, `expired`, `unknown`으로 구분 |
| `policy` | 기본 `unknown`, `is_expired=false` |
| `training` | 기본 `unknown`, `is_expired=false` |

마감 필터링은 현재 `startup_notice`에 우선 적용한다. `policy`, `training`은 구조화된 모집 종료일의 의미가 다를 수 있으므로 strict freshness 필터에서 과도하게 제외하지 않는다.

### 5.8 오류 응답

| 상태 | 발생 조건 | 응답 예시 |
|---|---|---|
| 400 | 메시지 2자 미만 | `{"detail": "질문이 너무 짧습니다. 구체적으로 입력해 주세요."}` |
| 422 | 필드 누락, `top_k` 범위 초과 | `{"detail": "요청 형식이 올바르지 않습니다.", "errors": [...]}` |
| 500 | 예상치 못한 예외 | `{"detail": "서버 내부 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."}` |
| 503 | 서버 메모리 부족 또는 데이터 준비 안 됨 | `{"detail": "서버 리소스가 부족합니다. 잠시 후 다시 시도해 주세요."}` |

## 6. 통합 지원 항목 목록 조회

### `GET /api/policies`

`data/processed/opportunities.json`을 기준으로 통합 지원 항목 목록을 반환한다.

현재 코드는 `page/page_size`가 아니라 `limit/offset` 방식을 사용하므로 본 명세도 코드에 맞춰 `limit/offset` 기준으로 작성한다.

### 6.1 쿼리 파라미터

| 파라미터 | 타입 | 필수 | 기본값 | 설명 |
|---|---|---|---|---|
| `limit` | `int` | 선택 | `20` | 반환 개수. 1~100 |
| `offset` | `int` | 선택 | `0` | 시작 위치. 0 이상 |
| `source_category` | `string` | 선택 | `null` | `policy`, `startup_notice`, `training` |
| `domain` | `string` | 선택 | `null` | 도메인 필터 |
| `keyword` | `string` | 선택 | `null` | 제목/요약/지원내용 키워드 검색 |

프론트에서 페이지 번호 방식으로 쓰고 싶다면 다음과 같이 변환한다.

```text
limit = page_size
offset = (page - 1) * page_size
```

### 6.2 요청 예시

```http
GET /api/policies?source_category=policy&keyword=청년수당&limit=20&offset=0
```


## 7. 통합 지원 항목 상세 조회

### `GET /api/policies/{item_id}`

`item_id` 기준으로 단일 지원 항목의 전체 필드를 반환한다.

현재 코드의 함수 인자명이 `policy_id`로 남아 있어도 실제 전달 값은 `item_id`이다. Swagger 혼선을 줄이기 위해 최종적으로는 path 변수명도 `{item_id}`로 맞추는 것을 권장한다.

### 7.1 경로 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `item_id` | `string` | 필수 | 예: `policy_20260605005400113228` |

### 7.2 요청 예시

```http
GET /api/policies/policy_20260605005400113228
```

## 8. 프론트엔드 표시 규칙

| 상황 | 처리 방식 |
|---|---|
| `source_category=policy` | 뱃지: 청년정책 |
| `source_category=startup_notice` | 뱃지: 창업지원 |
| `source_category=training` | 뱃지: 교육·훈련 |
| `application_url` 없음 | 신청 버튼 숨김 |
| `source_url` 있음 | 출처 보기 버튼 표시 |
| `needs_detail_check=true` | 카드 하단에 `원문 확인 필요` 표시 |
| `deadline_status=expired` | 기본적으로 숨김 또는 `마감` 뱃지 표시 |
| `deadline_status=open` | `모집 중` 또는 `신청 가능` 뱃지 표시 |
| `deadline_status=unknown` | 마감 정보 미확인 표시 |
| `info_score` | 사용자에게 추천 점수처럼 표시하지 않음 |
| `required_documents=null` | 제출 서류 영역 숨김 또는 `원문 확인 필요` 표시 |

## 9. 구현 확인 체크리스트

| 항목 | 확인 내용 |
|---|---|
| `chat_schema.py` | `PolicyRecommendation`에 `item_id`, `title`, `policy_id`, `policy_name`, `application_url` 포함 |
| `rag_chat_service.py` | workflow 결과의 `item_id/title/application_url`이 `/api/chat` 응답까지 보존 |
| `rag_service.py` | `deadline_status` 기본값이 `unknown`이고, `application_end_date` 결측이 `None/null`인지 확인 |
| `rag_service.py` | strict freshness 필터가 `startup_notice`에 우선 적용되고, `policy/training`을 과도하게 제외하지 않는지 확인 |
| `policies.py` | `/api/policies`가 `limit/offset` 기준으로 동작 |
| `policy_schema.py` | 목록/상세 응답에 `item_id/title`과 기존 alias를 함께 둘지 확인 |
| `policy_repository.py` | 실제 조회 key가 `item_id`인지 확인 |
| 문서 | `/api/policies`를 `page/page_size`가 아닌 `limit/offset` 기준으로 설명 |

## 10. 현재 기준 주의사항

| 구분 | 현재 기준 |
|---|---|
| `/api/chat` 추천 ID | `item_id` 우선, `policy_id` alias 동시 제공 |
| `/api/chat` 추천 제목 | `title` 우선, `policy_name` alias 동시 제공 |
| `/api/chat` 신청 URL | `application_url` 보존 |
| `/api/policies` 페이징 | 코드 수정 없이 `limit/offset` 기준 사용 |
| 마감 상태 | `startup_notice` 중심으로 `open/expired/unknown` 처리 |
| `UserConditions` 확장 | `region_code`, `keywords`는 현재 필수 계약에서 제외 |
