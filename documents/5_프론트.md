# 프론트엔드 구현 현황

작성 기준: 현재 저장소 코드 기준

현재 문서 버전: **ver 2.3**

---

## 1. 프로젝트 개요

### 프로젝트명

**이젠, 안심**

### 목적

청년 정책 정보를 한곳에서 검색하고, AI 상담, 커뮤니티, 뉴스, 영상, 알림, 마이페이지 기능을 통해 사용자가 필요한 정책과 정보를 쉽게 확인할 수 있도록 만든 서비스입니다.

### 사용자 주요 흐름

- 홈 화면에서 추천 정책, 정책 뉴스, 인기 검색어를 확인합니다.
- 정책명, 설명, 나이, 분야, 지역, 상태, 소득조건을 조합해 정책을 검색합니다.
- 검색 조건이 없을 때는 기본 20개 정책 목록을 자동으로 보여주지 않고, 검색 결과 없음 상태를 표시합니다.
- 정책 상세 화면에서 지원 대상, 신청 방법, 기간, 링크 등 상세 정보를 확인합니다.
- 로그인 사용자는 정책을 스크랩하고 마이페이지에서 관리합니다.
- AI 챗봇에 청년 정책 관련 질문을 입력하고 추천 정책을 확인합니다.
- 커뮤니티에서 게시글을 작성, 조회, 수정, 삭제합니다.
- 회원가입 시 이메일 인증번호 확인 후 가입합니다.
- 로그인 화면에서 인증했던 이메일로 비밀번호 찾기 및 재설정을 진행합니다.
- 마이페이지에서 비밀번호를 변경할 수 있습니다.
- 로그인 시 비밀번호를 일정 횟수 이상 틀리면 이메일 인증 기반 비밀번호 재설정으로 유도합니다.
- 헤더 알림 버튼에서 확인하지 않은 알림 수를 확인하고, 버튼 클릭 시 알림 목록을 확장해 볼 수 있습니다.

## 2. ver 2.3 개선사항 요약

### 브랜딩 및 공통 UI

- Header 우측 버전 배지를 `ver 2.3`으로 변경했습니다.
- 주요 메뉴 순서를 `홈 → AI 챗봇 → 정책 검색 → 뉴스 → 영상 → 커뮤니티`로 변경했습니다.
- 영상 페이지와 커뮤니티 페이지의 자체 배경색을 제거하고 전역 배경색을 따르도록 정리했습니다.
- 페이지별 배경 색상 차이로 생기던 화면 단차를 제거했습니다.

### Home

- 홈 히어로 검색창의 이동 대상을 정책 검색 페이지에서 `AI 안쉼 챗봇`으로 변경했습니다.
- 홈 검색창에 질문을 입력하면 `/chat?message=...`로 이동하고, 챗봇 페이지에서 해당 질문을 자동 전송합니다.
- 홈 추천 질문 칩도 정책 검색이 아니라 챗봇 질문으로 연결되도록 변경했습니다.

### 커뮤니티

- 커뮤니티 상단의 개발/관리용 표시 요소를 제거했습니다.
  - `REQ-F-09` 배지
  - `공개` 버튼
  - `MVT` 버튼
- 커뮤니티 페이지 폰트를 전역 폰트 설정을 따르도록 정리했습니다.
- 커뮤니티 목록 화면은 제목, 검색/필터, 게시글 요약, 목록 테이블 중심으로 단순화했습니다.

### 영상

- 영상 페이지의 배경을 커뮤니티 및 전체 레이아웃과 동일한 전역 배경으로 맞췄습니다.
- 로딩, 오류, 빈 목록 상태에서도 같은 배경 톤이 유지되도록 정리했습니다.

## 3. ver 2.2 개선사항 요약

### 브랜딩 및 공통 UI

- Header 우측 버전 배지를 `ver 2.2`로 변경했습니다.
- 로그인, 회원가입, 비밀번호 찾기 등 인증 관련 화면 상단의 텍스트 로고를 `/home_logo.png` 이미지 로고로 교체했습니다.
- 브라우저 탭 favicon을 `/home_logo.png`로 연결했습니다.

### 정책 검색

- 정책 검색 페이지 최초 진입 시 전체 정책 20개가 기본으로 노출되던 동작을 제거했습니다.
- 검색어와 필터 조건이 모두 비어 있으면 정책 API를 호출하지 않습니다.
- 초기 상태에서는 `검색 결과 없음`과 `검색어를 입력하거나 필터를 선택해 주세요` 안내를 표시합니다.
- 홈/인기 검색어 등에서 `keyword` URL 파라미터로 진입한 경우에는 기존처럼 자동 검색합니다.
- 기존 필터 구조인 `분야`, `지역`, `상태`, `소득조건`을 유지했습니다.
- 기존 필터에 `나이` 필터를 추가하고 정책 API `age` 파라미터로 전달하도록 연결했습니다.
- 필터 UI를 `조건 입력 필터` 스타일로 개편했습니다.
- 필터 영역은 별도 카드 상자 느낌을 줄이고, 페이지 배경과 자연스럽게 어우러지도록 조정했습니다.
- `조건 적용` 버튼을 통해 draft 필터를 실제 검색 조건에 반영합니다.
- `초기화` 버튼은 경계가 명확히 보이도록 테두리, 배경, 그림자 스타일을 적용했습니다.
- `조건 입력 팁`은 필터 그룹 아래로 간격을 두고 배치했으며, 리스트 정렬과 줄바꿈을 자연스럽게 정리했습니다.

### 커뮤니티

- 게시글 작성자 표시는 이메일 대신 닉네임 우선으로 표시합니다.
- `author_name`이 없을 때만 기존 작성자 값으로 fallback합니다.
- 게시글 카테고리를 추가했습니다.
  - `general`: 일반
  - `housing`: 주거
  - `finance`: 금융
  - `employment`: 취업
  - `education`: 교육
  - `startup`: 창업
  - `etc`: 기타
- 글 작성/수정 화면에서 카테고리를 선택할 수 있습니다.
- 커뮤니티 목록 필터를 전체 및 7개 카테고리 기준으로 변경했습니다.
- 게시글 수정/삭제 버튼은 본인이 작성한 글에서만 보이도록 처리했습니다.

### 인증 및 보안

- 회원가입 시 이메일 인증번호 발송/확인 과정을 추가했습니다.
- 이메일 인증이 완료되기 전에는 회원가입 버튼을 사용할 수 없습니다.
- 로그인 화면에 비밀번호 찾기 링크를 추가했습니다.
- `/forgot-password` 페이지에서 이메일 인증 후 새 비밀번호를 설정할 수 있습니다.
- 비밀번호 찾기 화면의 깨진 한글 문구와 에러 메시지 표시를 정리했습니다.
- 마이페이지에 비밀번호 변경 화면(`/mypage/password`)을 추가했습니다.
- 로그인 시 비밀번호를 설정 횟수 이상 틀리면 `PASSWORD_RESET_REQUIRED` 상태를 감지해 비밀번호 재설정으로 유도합니다.
- 비밀번호 재설정 화면은 로그인 실패에 사용한 이메일을 전달받아 초기값으로 사용합니다.

### 알림

- 기존 별도 알림 페이지(`/notifications`)는 제거했습니다.
- Header 알림 버튼을 누르면 현재 페이지 안에서 알림 목록이 드롭다운으로 확장됩니다.
- 알림 배지는 실제 확인하지 않은 알림 수만 표시합니다.
- 미확인 알림이 0개면 배지를 숨깁니다.
- 미확인 알림이 99개를 넘으면 `99+`로 표시합니다.
- 드롭다운에서 최근 알림 확인, 읽음 처리, 전체 읽음, 삭제가 가능합니다.
- 알림 링크가 있는 항목은 클릭 시 해당 링크로 이동하고 읽음 처리합니다.
- 마이페이지 알림 영역은 기존 구조를 유지하되 별도 알림 페이지 이동 링크는 제거했습니다.

## 6. API 연동 현황

### 인증

| 기능 | 프론트 서비스 | 비고 |
| --- | --- | --- |
| 로그인 | `authApi.login` | 실패 제한 상태 코드 처리 |
| 회원가입 | `authApi.signup` | 이메일 인증번호 포함 |
| 이메일 인증번호 발송 | `authApi.sendVerificationCode` | 회원가입/비밀번호 재설정 |
| 이메일 인증번호 확인 | `authApi.verifyEmailCode` | 인증 완료 상태 반영 |
| 비밀번호 재설정 | `authApi.resetPassword` | 인증 후 새 비밀번호 설정 |
| 비밀번호 변경 | `authApi.changePassword` | 로그인 사용자 전용 |
| 내 정보 조회 | `authApi.getMe` | 토큰 기반 |
| 프로필 수정 | `authApi.updateProfile` | 마이페이지 |

### 정책

| 기능 | 프론트 서비스 | 비고 |
| --- | --- | --- |
| 정책 목록 | `policyApi.getPolicies` | 검색 조건이 있을 때만 호출 |
| 정책 상세 | `policyApi.getPolicyDetail` | 상세 화면 |
| 정책 스크랩 | `policyApi.toggleScrap` | 로그인 사용자 |
| 스크랩 목록 | `policyApi.getScrapedPolicies` | 마이페이지 |
| 최근 본 정책 | `policyApi.getViewedPolicies` | 마이페이지 |
| 인기 검색어 | `policyApi.getPopularSearches` | 홈/검색 |

정책 목록 요청에는 `keyword`, `region`, `sourceCategory`, `age` 조건을 전달할 수 있습니다.

### 커뮤니티

| 기능 | 프론트 서비스 | 비고 |
| --- | --- | --- |
| 게시글 목록 | `communityApi.getPosts` | 카테고리 필터 |
| 게시글 상세 | `communityApi.getPost` | 작성자 닉네임 표시 |
| 게시글 작성 | `communityApi.createPost` | 카테고리 포함 |
| 게시글 수정 | `communityApi.updatePost` | 본인 글에서만 UI 노출 |
| 게시글 삭제 | `communityApi.deletePost` | 본인 글에서만 UI 노출 |

### 알림

| 기능 | 프론트 서비스 | 비고 |
| --- | --- | --- |
| 알림 목록 | `notificationApi.getNotifications` | Header 드롭다운에서 사용 |
| 미확인 수 | `notificationApi.getUnreadCount` | 배지 표시 |
| 단일 읽음 | `notificationApi.markAsRead` | 드롭다운 액션 |
| 전체 읽음 | `notificationApi.markAllAsRead` | 드롭다운 액션 |
| 알림 삭제 | `notificationApi.deleteNotification` | 드롭다운 액션 |

알림 API 경로는 유지하지만, 프론트 별도 알림 페이지는 사용하지 않습니다.

## 10. 백엔드 협업 필요 사항

- 커뮤니티 게시글 응답에 `author_name`, `author_id`, `category`, `category_label`이 안정적으로 포함되어야 합니다.
- 커뮤니티 `CommunityPost.category` 필드와 마이그레이션이 배포 DB에 적용되어야 합니다.
- 회원가입 이메일 인증번호 발송/검증 API가 운영 SMTP 설정과 연결되어야 합니다.
- 비밀번호 재설정용 인증번호 발송/검증 API가 운영 SMTP 설정과 연결되어야 합니다.
- 로그인 실패 제한은 운영 환경에서 캐시 또는 Redis 기반으로 관리하는 것이 권장됩니다.
- 알림 API는 미확인 알림 수, 읽음 처리, 전체 읽음, 삭제 기능을 유지해야 합니다.
- 정책 검색 나이 조건은 프론트에서 `age` 파라미터로 전달하므로 백엔드 필터 기준과 응답 범위를 확인해야 합니다.
