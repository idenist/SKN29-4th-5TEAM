## 1. 프로젝트 개요

### 프로젝트명

**이젠, 안쉼**

### 목적

청년 정책 정보를 한 곳에서 탐색하고, AI 상담, 정책 검색, 스크랩, 커뮤니티, 알림, 마이페이지 기능을 통해 사용자가 자신에게 필요한 정책 정보를 쉽게 확인할 수 있도록 돕는 웹 서비스입니다.

### 사용자가 할 수 있는 일

- 홈 화면에서 서비스 소개, 통계, 추천 정책, 정책 뉴스, 인기 검색어를 확인할 수 있습니다.
- 정책명, 지역, 분야 조건으로 정책을 검색하고 상세 정보를 확인할 수 있습니다.
- 로그인 상태에서 정책을 스크랩하고 마이페이지에서 관리할 수 있습니다.
- AI 안쉼 챗봇에 청년 정책 관련 질문을 입력하고 추천 정책 답변을 받을 수 있습니다.
- 커뮤니티 게시글 목록과 상세 글을 확인할 수 있습니다.
- 로그인 사용자는 커뮤니티 글 작성, 수정, 삭제를 할 수 있습니다.
- 커뮤니티 글 작성자는 이메일이 아니라 닉네임으로 표시됩니다.
- 커뮤니티 글 작성/수정 시 카테고리를 선택할 수 있습니다.
- 회원가입 시 이메일 인증번호 확인 후 가입할 수 있습니다.
- 로그인 화면에서 인증했던 이메일로 비밀번호 찾기/재설정을 할 수 있습니다.
- 로그인 사용자는 마이페이지에서 프로필, 스크랩 정책, 최근 본 정책, 검색 기록, 알림 요약을 확인할 수 있습니다.
- 로그인 사용자는 알림 목록 확인, 읽음 처리, 삭제를 할 수 있습니다.
- 뉴스와 영상 페이지는 현재 페이지 내부 목데이터 기반으로 표시됩니다.

## 2. ver 2.2 변경사항

### UI/브랜딩

- Header 우측 버전 배지를 `ver 2.1`에서 `ver 2.2`로 변경했습니다.
- 로그인/회원가입 인증 레이아웃 상단의 `EA` 텍스트 마크를 `/home_logo.png` 로고 이미지로 교체했습니다.
- 브라우저 탭 파비콘을 `/home_logo.png`로 연결했습니다.

### 커뮤니티

- 게시글 목록/상세 화면에서 작성자를 `author_email`이 아니라 `author_name` 우선으로 표시합니다.
- `author_name`이 없을 때만 이메일 또는 기존 author 값으로 fallback합니다.
- 게시글 카테고리 코드를 추가했습니다.
  - `general`: 일반
  - `housing`: 주거
  - `finance`: 금융
  - `employment`: 취업
  - `education`: 교육
  - `startup`: 창업
  - `etc`: 기타
- 커뮤니티 목록 필터를 전체 + 7개 카테고리 기준으로 변경했습니다.
- 글 작성/수정 모달에 카테고리 선택 필드를 추가했습니다.
- 게시글 카드와 상세 화면에는 `category_label` 또는 프론트 매핑 라벨을 표시합니다.

### 인증

- 회원가입 화면에 이메일 인증번호 발송/확인 흐름을 추가했습니다.
- 회원가입 API 요청에 `verification_code`를 포함하도록 변경했습니다.
- 이메일 인증 완료 전에는 회원가입 버튼이 비활성화됩니다.
- 로그인 화면에 `비밀번호를 잊으셨나요?` 링크를 추가했습니다.
- `/forgot-password` 페이지를 추가했습니다.
- 비밀번호 찾기 페이지에서 가입 이메일로 인증번호를 받고 새 비밀번호를 설정할 수 있습니다.

### 백엔드 연동 반영

- 커뮤니티 `CommunityPost.category` 필드와 마이그레이션을 추가했습니다.
- 커뮤니티 API 응답에 `author_name`, `category`, `category_label`을 사용하도록 프론트 어댑터를 갱신했습니다.
- 이메일 인증번호 저장용 `EmailVerificationCode` 모델과 마이그레이션을 추가했습니다.
- 인증번호 발송/확인, 비밀번호 재설정 API와 프론트를 연결했습니다.
- 개발 환경 기본 이메일 백엔드는 콘솔 출력이며, SMTP 환경변수 설정 시 실제 발송으로 전환할 수 있습니다.

## 6. 실행 방법

### Frontend 실행

```bash
cd frontend
npm install
npm run dev
```

기본 개발 서버:

```text
http://localhost:3000
```

기본 API 주소:

```text
http://127.0.0.1:8000/api
```

`frontend/src/services/apiClient.js` 기준으로 `VITE_API_BASE_URL` 환경변수가 없으면 위 기본 주소를 사용합니다.

### Frontend 빌드

```bash
cd frontend
npm run build
```

### Backend 실행

```bash
cd backend
python manage.py runserver
```

이 프로젝트 작업 환경에서는 다음 Python으로 백엔드 검증을 수행했습니다.

```bash
C:\Users\nowis\miniconda3\envs\skn\python.exe manage.py check
```

### Docker 실행

```bash
docker compose up --build
```

현재 `docker-compose.yml` 포함 서비스:

- `db`: PostgreSQL 16 alpine
- `backend`: Django/Gunicorn
- `nginx`: API, admin, static 프록시

주의: `frontend` 컨테이너는 현재 `docker-compose.yml`에서 주석 처리되어 있습니다.

## 9. API 연동 현황

### 공통 API 클라이언트

파일: `frontend/src/services/apiClient.js`

- Axios 인스턴스 사용
- 기본 API 주소: `http://127.0.0.1:8000/api`
- `VITE_API_BASE_URL`로 API 주소 변경 가능
- `accessToken`이 localStorage에 있으면 `Authorization: Bearer <token>` 헤더 추가
