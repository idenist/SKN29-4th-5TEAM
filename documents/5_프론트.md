# 프로젝트 진행 현황

작성 기준: 현재 저장소 코드 기준

이 문서는 팀원이 현재까지 구현된 기능과 프로젝트 구조를 빠르게 이해할 수 있도록 작성한 진행 현황 문서입니다. 구현되지 않았거나 코드에서 확인되지 않은 기능은 완료 기능으로 기록하지 않았습니다.

현재 문서 버전: **ver 2.1**

---

## 1. 프로젝트 소개

### 프로젝트명

**이젠, 안쉼**

### 프로젝트 목적

청년 정책 정보를 한 곳에서 탐색하고, AI 상담, 정책 검색, 스크랩, 커뮤니티, 알림, 마이페이지 기능을 통해 사용자가 자신에게 필요한 정책 정보를 더 쉽게 확인할 수 있도록 돕는 웹 서비스입니다.

### 사용자가 할 수 있는 기능

- 홈 화면에서 서비스 소개, 통계, 이용 흐름, 추천 정책, 정책 뉴스를 확인할 수 있습니다.
- 정책명, 지역, 분야 조건으로 정책을 검색할 수 있습니다.
- 정책 상세 정보를 확인하고 로그인 상태에서 정책을 스크랩할 수 있습니다.
- AI 챗봇에 청년 정책 관련 질문을 입력하고 추천 정책 응답을 받을 수 있습니다.
- 커뮤니티 게시글 목록을 보고, 상세 글을 확인할 수 있습니다.
- 로그인 사용자는 커뮤니티 글 작성, 수정, 삭제를 시도할 수 있습니다.
- 로그인, 회원가입, JWT 기반 인증 상태 유지가 구현되어 있습니다.
- 로그인 사용자는 마이페이지에서 프로필, 스크랩 정책, 최근 본 정책, 검색 기록, 알림 요약을 확인할 수 있습니다.
- 로그인 사용자는 알림 목록 확인, 읽음 처리, 삭제를 할 수 있습니다.
- 로그인 사용자는 프로필 정보와 프로필 이미지를 수정할 수 있습니다.
- 뉴스와 영상 페이지는 현재 페이지 내부 목데이터 기반으로 표시됩니다.
- 홈 검색창, 추천 검색어, 인기 검색어를 통해 정책 검색 페이지로 이동하고 검색 결과를 확인할 수 있습니다.

## 5. 실행 방법

### Frontend 실행

```bash
cd frontend
npm install
npm run dev
```

프론트 기본 개발 서버:

```text
http://localhost:3000
```

프론트 API 기본 주소:

```text
http://127.0.0.1:8000/api
```

`frontend/src/services/apiClient.js` 기준으로 `VITE_API_BASE_URL` 환경 변수가 없으면 위 기본 주소를 사용합니다.

### Frontend 빌드

```bash
cd frontend
npm run build
```

### Backend 실행

```bash
cd backend
python manage.py runserver
```

현재 로컬 환경에서 Django가 일반 `python`에 설치되어 있지 않을 수 있습니다. 이 프로젝트 작업 환경에서는 다음 Python으로 테스트 실행이 확인되었습니다.

```bash
C:\Users\nowis\miniconda3\envs\skn\python.exe manage.py test tests.test_community
```

### Docker 실행

프로젝트 루트에서 실행합니다.

```bash
docker compose up --build
```

현재 `docker-compose.yml` 기준 포함 서비스:

- `db`: PostgreSQL 16 alpine
- `backend`: Django/Gunicorn
- `nginx`: API, admin, static 프록시

주의: `frontend` 컨테이너는 `docker-compose.yml`에 주석 처리되어 있습니다. Nginx `/` 경로도 현재 프론트로 프록시하지 않고 기본 텍스트 응답을 반환하도록 되어 있습니다.

### 필수 패키지

#### Frontend

- React

- @vitejs/plugin-react

#### Backend

### Node 버전

`frontend/package.json`에 `engines` 필드는 정의되어 있지 않습니다.

### Python 버전

- `backend/Dockerfile`은 `python:3.11-slim` 이미지를 사용합니다.
- 로컬 실행 시 사용 Python 버전은 환경에 따라 다를 수 있습니다.

### Docker 사용 여부

- Docker Compose 설정이 존재합니다.
- DB, Backend, Nginx 서비스가 구성되어 있습니다.
- Frontend 컨테이너는 현재 주석 처리 상태입니다.

### 실행 시 주의사항

- 프론트는 기본적으로 `http://127.0.0.1:8000/api`로 API 요청을 보냅니다.
- 백엔드는 PostgreSQL 연결을 필요로 합니다.
- Docker Compose 사용 시 DB 환경 변수 기본값은 `docker-compose.yml`과 `settings.py`에 서로 다르게 정의된 부분이 있으므로 `.env` 설정을 확인해야 합니다.
- 로그인 필요한 페이지는 `ProtectedRoute`로 보호됩니다.
- 뉴스와 영상은 현재 실제 백엔드 API 연동이 아니라 페이지 내부 목데이터입니다.
- Nginx는 현재 React 프론트엔드를 서빙하지 않습니다.
