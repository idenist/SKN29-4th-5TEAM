## 6. 발견 이슈
[cite_start]통합 및 연동 테스트 단계에서 발견된 핵심 결함 항목은 총 3건이며, 모두 프론트엔드 및 인프라 파이프라인 정제 단계에서 식별되었습니다[cite: 103, 104].

* [cite_start]**[이슈 #01] 네이버 뉴스 원문 태그 노출 현상 (한예나 발견):** `NewsPage.jsx` 연동 시 네이버 Search API의 raw 데이터 제목 및 요약문 텍스트에 `<b>청년도약계좌</b>` 및 `&quot;` 같은 HTML 마크업 태그가 그대로 렌더링되어 UI 가독성을 저해하는 현상 발견[cite: 1117].
* [cite_start]**[이슈 #02] 외부 유튜브 iframe 재생 불안정 및 자동 재생 감점 리스크 (한예나 발견):** 일부 비디오 로드 실패 시 화면 전체 레이아웃이 공백 상태로 굳어버리는 결함이 식별되었으며, 일부 URL 주소 규격 뒤에 자동 재생 옵션(`autoplay=1`)이 개입되어 사용자 피로도를 유발할 리스크 감지[cite: 129, 132].
* **[이슈 #03] 7번 비디오 메타데이터 구문 분석 에러 (한예나 발견):** 백엔드 제출용 및 화면 렌더링용 `video_info.json` 파일 데이터 구성 중, 7번 데이터 타이틀 필드 내부에서 쌍따옴표(`"`)가 중복 선언되어 JSON 파일 전체 파싱 에러(`SyntaxError`)를 유발하는 결함 식별.

## 7. 수정 및 재검증 이력
[cite_start]발견된 3가지 이슈에 대해 QA 매니저 주도로 즉각적인 코드 보완 및 정합성 패치를 적용하여 최종 테스트를 통과시켰습니다[cite: 104].

* **[수정 #01 - 뉴스 태그 정제 패치]:** `NewsPage.jsx` 내부에 정규표현식 기반 문자열 정규화 헬퍼 함수 `cleanText()`를 전격 구축했습니다. `replace(/<\/?[^>]+(>|$)/g, "")` 구문을 통과시켜 HTML 태그를 완전 박멸하고 특수 문자를 문자열로 안전 치환하여 재검증 Pass 판정을 획득했습니다.
* **[수정 #02 - 비디오 Fallback UI 및 파라미터 규제]:** `VideoPage.jsx` 컨테이너 하위 구조에 CSS relative-absolute 트릭을 이용한 하단 백그라운드 레이어를 신설하고 `"현재 영상을 불러올 수 없습니다."` 텍스트를 고정 배치했습니다. [cite_start]또한 iframe `src` 매핑 로직에서 자동 재생 옵션을 원천 제외하고, `aspectRatio: '16/9'` 속성을 최우선 선언하여 디바이스 크기별 종횡비 유지를 완료했습니다[cite: 128, 129, 132].
* **[수정 #03 - JSON 타이틀 따옴표 교정]:** `video_info.json` 및 `VideoPage.jsx` 내부 소스코드 배열의 7번 아이템 타이틀 구문을 한국어 표준 따옴표인 `“매달 50만 원 준다고?”` 형태로 완전 교정하여 파싱 엔진 에러를 완전 청산했습니다.

## 8. 배포 환경 검증 결과
[cite_start]인프라 담당 정승 및 백엔드 팀과의 협업을 통해 AWS 클라우드 환경 배포 무결성을 검증하고 정상 동작을 확인했습니다[cite: 105, 1736].

* [cite_start]**인프라 아키텍처 연계 정합성:** Nginx(리버스 프록시)가 80/443 정적 요청을 처리하고 API 요청을 Gunicorn WSGI 매개체를 거쳐 Django 백엔드 서버 컨테이너로 안전하게 포워딩하는 구조를 현장 검증했습니다[cite: 994, 2014].
* [cite_start]**네트워크 접근 통제 (보안 그룹):** 외부 퍼블릭 망에서 AWS RDS 데이터베이스로의 다이렉트 5432 포트 접근은 원천 차단(`Drop`)됨을 확인했으며, 오직 EC2 가상 서버 인스턴스가 소속된 보안 그룹 ID(`Security Group`)의 인바운드 요청만 조건부 허용(`Accept`)하여 안전한 가두리 보안망을 구축했음을 검증했습니다[cite: 1129, 1377].
* [cite_start]**S3 미디어 스토리지 연동:** 마이페이지 프로필 사진 수정 폼 기능 수행 시, `uploadApi.js`를 거쳐 AWS S3 버킷에 유니크 식별자로 인코딩된 이미지 파일이 정상 생성됨을 확인하였고, 타 사용자가 해당 경로에 불법 쓰기(`Write`) 명령을 수행할 수 없도록 퍼블릭 쓰기 권한이 철저히 제한되어 있음을 확인했습니다[cite: 385, 1675, 1684].

## 9. Docker 재현성 검증
[cite_start]로컬 개발 자산의 실서버 이식 시 환경 파편화를 방지하기 위한 가상화 재현성을 검증했습니다[cite: 106].

* [cite_start]**멀티 컨테이너 오케스트레이션 검증:** `deployment/docker-compose.prod.yml` 스크립트를 실행하여 `frontend`, `backend`, `nginx` 3대 핵심 컨테이너가 격리된 가상 네트워크 환경 안에서 상호 충돌 없이 독립 구동됨을 체크했습니다[cite: 1795].
* **재기동 복구 및 데이터 영속성 테스트:** `docker compose down` 명령으로 전체 인프라 서비스를 물리적으로 완전히 종료한 후, 다시 `up -d` 명령을 통해 재실행을 단행했습니다. [cite_start]이 과정에서 PostgreSQL 컨테이너와 매핑된 볼륨 데이터의 DB Migration 유실 현상이 단 1건도 발생하지 않았으며, 기존 적재된 청년 정책 마이페이지 스크랩 데이터 셋이 원상태로 영속 복구됨을 확인하여 **100% 완전한 배포 재현성**을 증명했습니다[cite: 1651, 1714, 1715, 1717].

## 10. 최종 한계 및 개선 방향
[cite_start]3차 프로젝트에서 도출된 기술적 단점과 4차 통합 테스트 과정을 거치며 발견된 인프라 및 UI 레벨의 한계점과 이에 대한 실무적 개선 로드맵을 선언합니다[cite: 107, 2058].

* [cite_start]**[한계점 1] ReAct 에이전트 다중 호출로 인한 응답 지연(Latency):** 사용자의 조건(나이, 거주지 등)을 심사하는 `Eligibility Checker`와 데이터 품질을 점검하는 `Result Sufficiency Checker`가 LangGraph 워크플로우 상에서 반복 구동됨에 따라, 챗봇 질문 전송 시 최종 답변 생성까지의 지연시간(평균 5~8초)이 발생하는 UX적 제약 확인[cite: 2067, 2109].
  * [cite_start]**➡️ 개선 방향:** 실서비스 구동 시 사용자의 이탈을 방지하기 위해, 백엔드 API 단에서 SSE(Server-Sent Events) 프로토콜을 이용한 자연어 스트리밍 응답 출력을 5차 고도화 규격으로 반영하고, 프론트엔드 단에 실시간 추론 단계를 시각화하는 가이드 인디케이터 UI를 신설할 계획입니다[cite: 598, 602].
* [cite_start]**[한계점 2] 수치형 제약조건(Hard Constraints) 필터링 환각 가능성:** RAG 벡터 검색은 단순 텍스트 맥락 탐색에 특화되어 있어 "중위소득 120% 이하"와 같은 엄격한 수치 조건을 100% 걸러내지 못해 발생하는 미세한 정보 노이즈 잔존[cite: 2061, 2062].
  * [cite_start]**➡️ 개선 방향:** 백엔드 수집 단에서 정승님이 구축한 `deadline_status` 파싱 자동화 로직처럼 나이/소득 필드를 Django ORM 레벨에서 1차로 물리 필터링(Hard Filtering)하는 메타데이터 매칭 구조를 정교화하여, LLM 순수 추론 의존도를 낮추고 신뢰도를 99.9%까지 끌어올리겠습니다[cite: 875, 2078].
