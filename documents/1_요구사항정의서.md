# [이젠, 안쉼] 4차 프로젝트 요구사항 정의서

## 1. 프로젝트 개요
* **목표:** 청년 지원 정보 통합 탐색 에이전트의 모던 웹 전환(React + Django) 및 개인화/커뮤니티 기능 고도화
* **핵심 평가 대응:** 인증·권한 체계 구축, 비동기 LLM 연동 및 UX 처리, Flex/Grid 기반 반응형 UI, AWS/Docker 배포 환경 구성

---

## 2. 사용자 시나리오 (User Scenarios)
> 주요 유스케이스가 사용자 관점에서 타당하게 도출되었음을 증명합니다.

1. **[인증·권한 및 맞춤형 추천]** 사용자는 사이트에 접속하여 **회원가입/로그인**을 진행하고, 개인정보처리방침에 동의한 뒤 사용자 프로필(연령, 거주지, 관심분야)을 등록한다. 메인 화면에 진입하면 DB에 저장된 데이터를 바탕으로 맞춤형 정책이 추천된다.
2. **[LLM 연동 및 비동기 UX]** 사용자는 AI 챗봇에게 정책을 묻는다. 시스템은 **비동기 호출**을 통해 LLM 답변을 실시간 타자 효과로 제공하며, 대기 시간 동안 '에이전트 검색 중'이라는 로딩 스피너를 노출하여 이탈을 방지한다.
3. **[활동 및 커뮤니티]** 사용자는 '예상 지원금 안내' 페이지에서 최대 혜택을 확인하고, 관심 정책을 '스크랩'한다. 이후 마감/변경 알림을 받으며, 커뮤니티 게시판과 청년 뉴스 연동 메뉴를 통해 정보를 확장 탐색한다.

## 5. LLM API 연동(REQ-N-02) 세부 명세

### 5.1 호출 방식 및 통신 포맷
* **호출 위치:** React 프론트엔드 ➔ Django 백엔드 (/api/ai/chat/)
* **호출 방식:** Fetch API 또는 Axios 기반 비동기 호출 (async/await 사용)
* **요청 메서드/형식:** POST / application/json

### 5.2 요청/응답 예시

**요청(Request) JSON:**

    {
      "message": "서울에 사는 25세 청년이 받을 수 있는 월세 지원 정책 알려줘",
      "user_profile": {
        "age": 25,
        "region": "서울",
        "interests": ["주거", "복지"]
      },
      "top_k": 5,
      "conversation_id": null
    }

**응답(Response) JSON:**

    {
      "answer": "서울 거주 25세 청년이라면 청년월세지원 사업을 우선 확인할 수 있습니다. 다만 소득 조건과 신청 기간은 원문에서 추가 확인이 필요합니다.",
      "recommendations": [
        {
          "policy_id": "policy_001",
          "title": "청년월세지원",
          "summary": "청년의 주거비 부담 완화를 위한 월세 지원 정책",
          "eligibility": "추가 확인 필요",
          "application_url": "https://example.com/apply",
          "source_url": "https://example.com/source"
        }
      ],
      "sources": [
        {
          "title": "청년월세지원",
          "url": "https://example.com/source"
        }
      ],
      "warnings": [
        "실제 신청 가능 여부는 신청 기간과 소득 조건에 따라 달라질 수 있습니다."
      ],
      "error": null
    }

### 5.3 비동기 UX 및 예외 상황 처리 방안
| 상황 | 처리 방식 및 화면 대응 |
| :--- | :--- |
| **LLM 응답 대기 중** | 입력창 비활성화, 로딩 스피너 표시, "에이전트 검색 중" 문구 노출 |
| **LLM API 호출 실패** | "AI 답변 생성에 실패했습니다. 잠시 후 다시 시도해주세요." 메시지 표시 |
| **응답 지연/타임아웃** | 일정 시간 경과 시 HTTP 408 Timeout 처리 후 재시도 버튼 제공 |
| **검색 결과 없음** | 정책을 임의 생성(환각)하지 않고, "조건 변경 또는 공식 출처 확인" 안내 |
| **출처 URL 없음** | 답변에 단정적 표현을 피하고 "원문 확인 필요" 경고 문구 표시 |
