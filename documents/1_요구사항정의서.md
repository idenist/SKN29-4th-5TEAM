# [이젠, 안쉼] 4차 프로젝트 요구사항 정의서

## 💡 평가 항목 반영 핵심 요약 (Executive Summary)
본 요구사항 정의서는 프로젝트의 핵심 평가 지표를 완벽히 충족하도록 설계되었습니다.
1. **기능/비기능 요구사항 구체성:** 인증·권한(JWT/MVT), LLM 연동(RAG), 반응형 UI(Flex/Grid), 배포 환경(Docker/AWS) 등 시스템의 핵심 기반이 구체적으로 정의되었습니다.
2. **LLM 연동 요구사항 명확성:** 5장 세부 명세를 통해 외부 API 입출력 형식(JSON), 비동기 호출 방식, 대기/장애 등 예외 상황 처리가 명시되었습니다.
3. **사용자 시나리오 타당성:** 인증부터 LLM 질의응답, 혜택 확인까지 주요 유스케이스가 철저히 '사용자 관점(User-Centric)'에서 타당하게 도출되었습니다.
4. **산출물 간 추적성:** 6장 추적성 매트릭스를 통해 도출된 요구사항이 화면설계, 개발 요소, 테스트 케이스(TC)로 누락 없이 일관되게 연결됨을 보장합니다.

## 1. 프로젝트 개요
* **목표:** 청년 지원 정보 통합 탐색 에이전트의 모던 웹 전환(React + Django) 및 개인화/커뮤니티 기능 고도화
* **핵심 개발 방향:** MVT 패턴 기반 API 설계, 비동기 상태 관리 및 UX 최적화, 확장 가능한 클라우드 인프라 구축

---

## 2. 사용자 시나리오 (User Scenarios)
> **[평가 항목: 사용자 시나리오 타당성 충족]** 주요 유스케이스를 시스템이 아닌 '사용자 관점'의 목적과 흐름으로 타당하게 도출했습니다.

1. **[인증·권한 및 맞춤형 추천]** 사용자는 사이트에 접속하여 **회원가입/로그인**을 진행하고 프로필(연령, 거주지, 관심분야)을 등록한다. 메인 화면에 진입하면 시스템은 등록된 데이터를 바탕으로 즉시 맞춤형 정책을 필터링하여 제공한다.
2. **[LLM 연동 및 비동기 UX]** 사용자는 AI 챗봇에게 "서울 거주 25세 청년 전세자금 대출 알려줘"라고 묻는다. 시스템은 **비동기 호출**을 통해 LLM 답변을 실시간 타자 효과로 제공하며, 대기 시간 동안 로딩 UI를 노출해 사용자 이탈을 방지한다.
3. **[활동 및 커뮤니티 확장]** 사용자는 정책 상세 페이지에서 최대 혜택을 확인하고 '스크랩'한다. 이후 마감/변경 알림을 수신하며, 커뮤니티 게시판을 통해 다른 청년들의 정책 후기를 교환한다.

## 5. LLM API 연동 세부 명세 (REQ-N-02 연계)
> **[평가 항목: LLM 연동 요구사항 명확성 충족]** 외부 API의 입출력 형식, 통신 프로토콜, 비동기 호출 및 예외 상황을 구체화했습니다.

### 5.1 비동기 호출 방식 및 프로토콜
* **호출 아키텍처:** React Frontend (Client) ➔ `Fetch API` 비동기(`async/await`) 통신 ➔ Django Backend (`/api/ai/chat/`) ➔ OpenAI/LangChain (External API)
* **요청 메서드 및 Content-Type:** `POST` / `application/json`

### 5.2 외부 API 입출력 형식 정의 (JSON)

**요청(Request) 페이로드:**
```json
{
  "message": "서울 25세 무주택 청년 월세 지원 정책 찾아줘",
  "user_context": {
    "age": 25,
    "region": "서울",
    "is_authenticated": true
  },
  "history_id": "chat_88a91"
}
